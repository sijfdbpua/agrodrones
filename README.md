# 🚁 AgroDrones

**AgroDrones** — учебный Python-проект: симуляция дистанционного мониторинга поля и обработки проблемных участков.

Проект показывает, как данные с виртуальных датчиков превращаются в управляемые задачи: система находит проблемные клетки, расставляет приоритеты, строит маршрут и показывает результат в консоли или веб-интерфейсе.

---

## ✨ Что есть в проекте

- 🧠 **ООП-модель**: поле, датчики, зоны, база, разные типы дронов.
- 📊 **Алгоритмы**:
  - обход двумерной сетки поля;
  - сортировка зон по приоритету через merge sort;
  - построение маршрута жадным алгоритмом ближайшей точки.
- 🗂️ **Структуры данных**:
  - двумерный список для карты поля;
  - очередь задач на `deque`;
  - односвязный список для истории облётов.
- 🖥️ **Консольная версия** с меню.
- 🌐 **Веб-версия на Flask** с визуальной картой поля.

---

## 🧩 Структура проекта

```text
agrodrones/
├── algorithms.py          # сортировка зон и построение маршрута
├── index.html             # веб-интерфейс
├── main.py                # консольная точка входа
├── requirements.txt       # зависимости проекта
├── simulation.py          # движок симуляции
├── web_server.py          # Flask-сервер
└── models/
    ├── base.py            # база и история облётов
    ├── drone.py           # классы дронов и очередь задач
    ├── field.py           # поле как сетка датчиков
    ├── sensor.py          # виртуальный датчик
    └── zone.py            # проблемная зона / задача
```

## 🚀 Быстрый запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/sijfdbpua/agrodrones.git
cd agrodrones
```

### 2. Создать виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

---

## 🖥️ Консольная версия

```bash
python main.py
```

В меню можно:

1. показать карту поля;
2. прожить один рабочий день;
3. посмотреть статус техники;
4. открыть историю облётов;
5. начать новый цикл.

---

## 🌐 Веб-версия

```bash
python web_server.py
```

После запуска открой в браузере:

```text
http://localhost:5000
```

Веб-интерфейс показывает карту поля, проблемные зоны, обработанные участки, заряд, бак и результат рабочего дня.

## 🧠 Что можно сказать на защите

> Проект моделирует цифровой контур агромониторинга: датчики собирают данные о поле, система рассчитывает проблемность каждой клетки, сортирует зоны по приоритету и передаёт задачи рабочему дрону. В проекте используются ООП, наследование, инкапсуляция, очередь задач, связный список и несколько алгоритмов обработки данных.

## 🔬 Алгоритмическая часть

| Часть | Где находится | Сложность |
|---|---|---:|
| Обход сетки поля | `models/field.py` | `O(rows * cols)` |
| Сортировка зон merge sort | `algorithms.py` | `O(n log n)` |
| Жадный маршрут | `algorithms.py` | `O(k²)` |
| Очередь задач | `models/drone.py` | `O(1)` на извлечение |
| История облётов | `models/base.py` | `O(1)` на добавление |
